# Code of Conduct

Stupify is a small project with sharp opinions, but the collaboration should
stay useful, direct, and humane.

## Expected Behavior

- Be respectful of different experience levels, tools, backgrounds, and working
  styles.
- Critique code, behavior, and tradeoffs instead of attacking people.
- Keep reports and examples privacy-safe. Do not post private code, diffs,
  filenames, repo URLs, author names, or private package names without approval.
- Assume good intent when possible, and ask clarifying questions before
  escalating.
- Accept maintainer decisions about project scope, tone, and release timing.

## Unacceptable Behavior

- Harassment, threats, slurs, sexualized comments, or sustained personal attacks.
- Publishing private information without explicit permission.
- Deliberately derailing issues, PRs, or reviews.
- Encouraging unsafe use of the tool, including hidden upload boundaries or
  misleading claims about where code is analyzed.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or PRs; block
participants; or take other reasonable action to keep the project healthy.

If you need to report conduct concerns, contact the maintainer through GitHub.
Include relevant links or screenshots when safe to share.
